# 危化品运输行业 PRD 生成

## 触发场景

当用户要求为已生成的原型页面产出 PRD 时使用。例如：
- "为运输任务管理模块生成 PRD"
- "根据这些页面生成一份完整的 PRD"

## 核心理念

1. **一个 PRD 对应一个业务系统**，覆盖该系统的所有页面
2. **每个页面的需求编号独立从 1 开始**，不同页面间编号互不影响
3. **PRD 只写业务功能设计**，不描述 UI 实现

## 前置步骤

1. 确认业务模块范围和涉及页面
2. 读取所有涉及页面的 `index.vue` + `spec.md` + `design.md`
3. 读取 `skills/industry-data-rules.md` 行业数据规范

## PRD 文档结构

```markdown
# [系统名称] PRD

> 每个页面的需求编号独立从 1 开始。

## 一、页面A名称
**页面**：xxx | **类型**：看板/列表/表单/移动端 | **需求数**：N

### [1] 需求名称
需求描述...

### [2] 需求名称
...

## 二、页面B名称
...

## 权限规划
## 全局交互
```

## 核心规则

### PRD 只写业务，不写 UI

禁止包含：页面布局描述、CSS样式、组件实现细节、前端框架术语。

应该描述：功能模块的业务职责、字段的业务含义和约束、操作的业务规则和前置条件、异常场景的业务处理逻辑。

### 需求编号（每页独立）

- 每个页面的需求编号独立从 1 开始
- 同页面内编号唯一
- 编号格式 `[N]`，写在 PRD 对应需求描述起始位置
- 弹窗类需求独立编号，标注 `type=modal`

### 移动端内部子页面

移动端内部子页面按独立页面处理。即使多个手机页面写在同一个 `index.vue` 中，只要在 PhoneFrame 内存在首页、一级页面、二级页面、详情页、步骤页等切换状态，就必须在 PRD 中拆成多个页面章节。

拆分规则：

- 手机首页单独成页，例如 `移动端巡检首页`，页面标识对应 `mobile-inspection`。
- 一级/二级/详情/任务步骤单独成页，例如 `移动端开始巡检`、`移动端巡检任务`、`移动端巡检记录`、`移动端记录详情`。
- 每个移动端子页面编号独立从 `[1]` 开始，除非现有标注实现已经采用连续编号；如果采用连续编号，必须保证页面 DOM 的 `data-req-id` 与 PRD 完全一致。
- PRD 页面标题、页面标识、DOM `data-req-page` 三者必须能一一对应。
- 返回、进入详情、下一步、完成后跳转等页面切换行为必须在对应需求中说明其目标页面，避免标注面板无法判断当前子页面。

推荐对应关系：

| PRD章节 | DOM 页面标识 |
|--------|-------------|
| 移动端巡检首页 | `mobile-inspection` |
| 移动端开始巡检 | `mobile-inspection-start` |
| 移动端巡检任务 | `mobile-inspection-task` |
| 移动端巡检记录 | `mobile-inspection-records` |
| 移动端记录详情 | `mobile-inspection-detail` |

### 权限点

查看归【查看】、编辑归【编辑】、导出归【导出】。不定义具体角色。

### 行业数据

PRD 中示例数据必须符合 `skills/industry-data-rules.md`。

## 输出

项目根目录 `PRD.md` —— 单份文档覆盖全部页面。

## 工作流程

1. 确认业务模块范围和涉及页面
2. 读取所有页面文件
3. 按页面逐一提取功能模块、字段、操作、交互、状态、权限
4. 每个页面独立分配需求编号 [1] [2] [3]...
5. 生成统一 PRD 写入 `PRD.md`
6. 若页面为移动端多子页面，确认每个子页面都有独立章节和稳定页面标识
7. 自检：是否包含UI描述、编号是否唯一、模块内联动是否完整
