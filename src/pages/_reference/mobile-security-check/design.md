# 装货安全检查 — 设计规范

## 布局结构
PhoneFrame 容器内：自定义头部 → 任务信息卡片 → 检查项列表 → 异常说明（条件显示）→ 签名区 → 底部操作栏

## 组件层级
- PhoneFrame：375px 移动端外壳
- MobileHeader：返回 + 标题 + 进度角标
- TaskInfoCard：任务编号、车牌、化学品
- CheckList：8项检查条目（圆形 radio + 名称 + 描述）
- RemarkArea：不合格时出现的异常说明输入框
- SignaturePad：虚线框点击签名
- BottomActions：暂存 + 提交按钮
- SubmitModal：提交成功结果弹窗

## 交互状态

| 状态 | 表现 |
|------|------|
| 未检查 | radio 灰色空心圆 |
| 检查合格 | radio 绿色实心 + 白色勾 |
| 检查不合格 | radio 红色实心 + 白色叉 + 拍照按钮出现 |
| 未完成全部检查 | 提交按钮禁用态 + 显示剩余待检查数 |
| 提交成功 | 绿色对勾弹窗 |

## 视觉风格
- 外壳：PhoneFrame 黑色边框 + 圆角
- 内容：白底卡片 10px 圆角，间距 6-14px
- 按钮：橙黄提交键、灰色暂存键、14px 加粗
- 触控：检查项整行可点击区域，按钮最小高度 44px
