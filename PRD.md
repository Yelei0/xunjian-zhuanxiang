# 园区巡检与专项检查 PRD

> 本文档作为原型页面需求说明的数据源。每个页面的需求编号独立从 1 开始；移动端 PhoneFrame 内部子页面按独立页面处理。

## 一、巡检记录

**页面**：inspection-record
**类型**：后台列表
**菜单路径**：园区巡检 / 巡检记录
**页面说明**：用于查询、导出、查看和维护园区巡检记录，并支持查看巡检详情、路线和整改结果。
**需求数**：7

### 页面功能描述

巡检记录页面面向园区管理人员，用于回溯园区巡检任务的执行结果。页面覆盖记录筛选、数据导出、状态查看、详情回看、路线查看和删除维护，不处理真实后端数据。

### 页面状态机

| 状态 | 业务含义 | 进入条件 | 可执行操作 | 后续状态 |
|------|----------|----------|------------|----------|
| 未提交 | 巡检任务已创建但尚未提交结果 | 移动端暂存后生成 | 查看列表、进入移动端继续填写、删除 | 已提交 / 已完成 |
| 已提交 | 巡检结果已提交且存在异常点位待整改 | 提交巡检且存在异常点位 | 查看详情、查看路线、处理整改、删除 | 已完成 |
| 已完成 | 全部点位正常，或异常点位已完成整改 | 巡检全部正常或整改完成 | 查看详情、查看路线、删除 | 无 |

### 页面特殊情况

| 场景 | 触发条件 | 处理规则 |
|------|----------|----------|
| 空数据 | 当前筛选条件下没有巡检记录 | 展示空状态，允许用户重置筛选条件 |
| 未提交记录 | 记录状态为未提交 | 列表中不展示巡检结果摘要，详情中按未提交状态展示 |
| 未完成整改 | 已提交记录仍有异常点位未整改 | 整改完成时间显示为“-” |
| 删除取消 | 用户在删除确认弹窗中取消 | 关闭弹窗，不改变原列表数据 |

### [1] 筛选区 {type=inline, target=filter-section}

#### 功能说明

系统支持按巡检人、巡检场地、巡检开始时间和巡检类型组合筛选巡检记录。筛选条件为空时展示全部记录。

#### 字段规则

| 字段 | 类型 | 是否必填 | 默认值 | 长度/范围 | 校验规则 | 备注 |
|------|------|----------|--------|-----------|----------|------|
| 巡检人 | 文本输入 | 否 | 无 | 最多 20 字 | 支持按姓名关键字模糊匹配 | 示例：张扬洲、马伟杜 |
| 巡检场地 | 下拉选择 | 否 | 请选择 | 物流园区、接收站、物流园区&接收站 | 仅允许选择枚举值 | 与巡检记录所属场地匹配 |
| 巡检开始时间 | 日期范围 | 否 | 无 | 开始日期至结束日期 | 结束日期不得早于开始日期 | 按巡检开始日期过滤 |
| 巡检类型 | 下拉选择 | 否 | 请选择 | 日常巡检-白天、日常巡检-夜晚、专项巡检、应急巡检 | 仅允许选择枚举值 | 与记录巡检类型匹配 |

#### 操作规则

| 操作 | 触发位置 | 前置条件 | 成功结果 | 失败/异常处理 |
|------|----------|----------|----------|----------------|
| 修改筛选条件 | 筛选字段 | 无 | 更新待查询条件 | 不立即刷新列表 |
| 清空单项条件 | 筛选字段 | 当前字段有值 | 字段恢复默认值 | 无 |

### [2] 搜索/重置 {type=inline, target=filter-actions}

#### 功能说明

搜索和重置用于控制筛选条件生效。搜索按当前筛选条件刷新巡检记录列表，重置清空条件并恢复默认列表。

#### 操作规则

| 操作 | 触发位置 | 前置条件 | 成功结果 | 失败/异常处理 |
|------|----------|----------|----------|----------------|
| 搜索 | 筛选区按钮 | 日期范围合法 | 列表展示符合条件的记录 | 日期范围不合法时提示并阻止搜索 |
| 重置 | 筛选区按钮 | 无 | 清空巡检人、巡检场地、时间范围和巡检类型，恢复默认列表 | 无 |

### [3] 导出功能 {type=inline, target=export-action}

#### 功能说明

系统支持导出当前筛选条件下的巡检记录数据。原型阶段仅模拟导出行为，不生成真实文件。

#### 操作规则

| 操作 | 触发位置 | 前置条件 | 成功结果 | 失败/异常处理 |
|------|----------|----------|----------|----------------|
| 导出 | 筛选区工具栏 | 当前列表存在可导出记录 | 模拟导出当前筛选范围数据 | 当前无记录时提示暂无可导出数据 |

#### 数据规则

| 数据项 | 规则 |
|--------|------|
| 导出范围 | 遵循当前筛选条件；未设置筛选条件时导出全部记录 |
| 导出字段 | 与列表核心字段保持一致，包括状态、结果摘要和时间信息 |

### [4] 数据表格 {type=inline, target=data-table}

#### 功能说明

系统展示巡检记录清单，并按任务状态、巡检结果和时间信息帮助管理人员判断记录处理进度。

#### 列表字段

| 字段 | 字段说明 | 是否必显 | 默认排序 | 展示规则 |
|------|----------|----------|----------|----------|
| 序号 | 当前列表行号 | 是 | 否 | 按当前列表顺序从 1 开始 |
| 巡检场地 | 记录所属场地 | 是 | 否 | 物流园区、接收站或物流园区&接收站 |
| 巡检类型 | 本次巡检业务类型 | 是 | 否 | 按类型标签展示 |
| 巡检人 | 本次巡检执行人员 | 是 | 否 | 展示人员姓名 |
| 巡检点位 | 本次巡检涉及点位 | 是 | 否 | 多个点位使用顿号分隔 |
| 任务状态 | 当前记录处理状态 | 是 | 否 | 未提交、已提交、已完成 |
| 巡检结果 | 点位结果汇总 | 是 | 否 | 未提交不展示结果；正常、异常、已整改按语义展示 |
| 巡检开始时间 | 首个点位开始巡检时间 | 是 | 是 | 年-月-日 时:分:秒 |
| 巡检提交时间 | 巡检结果提交时间 | 是 | 否 | 未提交时显示“-” |
| 整改完成时间 | 最后一个异常点位完成整改时间 | 是 | 否 | 未完成整改时显示“-” |
| 操作 | 当前记录可执行操作 | 是 | 否 | 巡检详情、删除 |

#### 数据规则

| 数据项 | 规则 |
|--------|------|
| 任务状态 | 未提交表示尚未提交结果；已提交表示存在异常待整改；已完成表示全部正常或异常已整改 |
| 巡检结果摘要 | 全部正常；或  “存在异常 N¦已整改 M”   |
| 整改完成时间 | 取记录内最后一个整改打卡时间；没有整改记录时显示“-” |

### [5] 巡检详情 {type=modal, modalId=detailModal, trigger=req:4.action:view-detail}

#### 功能说明

用户点击“巡检详情”后，系统展示该条记录的完整巡检信息，用于回看巡检过程和整改结果。

#### 触发规则

| 触发入口 | 前置条件 | 打开内容 |
|----------|----------|----------|
| 巡检详情 | 当前记录存在 | 打开当前记录详情弹窗 |

#### 展示内容

| 区域 | 内容 | 规则 |
|------|------|------|
| 基础信息 | 巡检类型、巡检场地、巡检人、开始时间、任务状态 | 只读展示 |
| 结果摘要 | 全部正常或异常/已整改数量 | 未提交记录不展示结果摘要 |
| 巡检路线 | 按点位顺序生成路线图 | 可进入路线大图查看 |
| 点位详情 | 打卡时间、点位照片、巡检情况 | 按点位顺序展示 |
| 整改信息 | 整改打卡、整改照片、整改情况 | 仅异常或已整改点位展示 |

#### 操作规则

| 操作 | 前置条件 | 成功结果 | 失败/异常处理 |
|------|----------|----------|----------------|
| 关闭详情 | 无 | 关闭弹窗，不改变数据 | 无 |
| 查看路线 | 当前记录存在点位 | 打开路线大图弹窗 | 无点位时提示暂无路线 |

### [6] 巡检路线查看 {type=modal, modalId=routeModal, trigger=req:5.action:view-route}

#### 功能说明

用户点击巡检路线入口后，系统单独展示当前记录的巡检路线，用于回看本次巡检路径。

#### 展示内容

| 内容 | 规则 |
|------|------|
| 起点 | 路线必须从起点开始 |
| 点位顺序 | 按巡检点位顺序依次连接 |
| 连接方式 | 不同点位之间使用横线和竖线连接，不允许斜线直连，顺序下右上左 |
| 行进方向 | 路线中展示方向提示 |

### [7] 删除功能 {type=modal, modalId=deleteModal, trigger=req:4.action:delete}

#### 功能说明

用户删除巡检记录前必须二次确认，避免误删历史巡检结果。

#### 操作规则

| 操作 | 前置条件 | 成功结果 | 失败/异常处理 |
|------|----------|----------|----------------|
| 确认删除 | 用户确认 | 从当前列表移除该记录 | 删除失败时保留原记录并提示 |
| 取消 | 无 | 关闭弹窗，不改变原数据 | 无 |

## 二、巡检类型

**页面**：inspection-type
**类型**：后台列表
**菜单路径**：园区巡检 / 巡检类型
**页面说明**：用于维护巡检类型及其适用场地，供移动端发起巡检时选择。
**需求数**：5

### 页面功能描述

巡检类型页面面向园区管理人员，用于维护可发起的巡检业务类型。巡检类型必须关联应用场地，后续移动端根据类型自动匹配场地。

### 页面特殊情况

| 场景 | 触发条件 | 处理规则 |
|------|----------|----------|
| 空数据 | 当前无巡检类型或筛选无结果 | 展示空状态 |
| 保存缺失 | 类型名称或应用场地为空 | 阻止保存并提示补全 |
| 删除取消 | 用户取消删除 | 关闭弹窗，不改变列表 |

## 三、巡检点位

**页面**：inspection-point
**类型**：后台列表
**菜单路径**：园区巡检 / 巡检点位
**页面说明**：用于维护巡检点位、点位坐标和点位码，支撑移动端点位选择、地图展示和扫码核对。
**需求数**：7

### 页面功能描述

巡检点位页面面向园区管理人员，用于配置物流园区和接收站的巡检点位。点位坐标用于地图展示和路线回放，点位码用于移动端巡检核对。

### 页面状态机

无独立状态机，页面仅维护巡检点位配置记录。

| 场景 | 触发条件 | 处理规则 |
|------|----------|----------|
| 坐标未选 | 新增或编辑时未选择地图坐标 | 阻止保存并提示选择坐标 |
| 点位码查看 | 用户查看点位码 | 打开点位码弹窗，原型阶段仅展示模拟码 |
| 删除取消 | 用户取消删除 | 关闭弹窗，不改变列表 |

## 全局交互

1. 所有删除操作必须二次确认。
2. 所有表单保存前必须校验必填项。
3. 移动端巡检任务完成前必须校验所有点位的打卡、照片和巡检情况。
4. 弹窗类操作取消时不改变原数据。
5. 数据刷新、提交、暂存、导出均为前端模拟行为，不对接真实后端。
6. 照片、扫码、点位码下载均为原型模拟能力，不接入真实设备或文件服务。
